0. Data extraction¶

We try to extract the data in Google Patents Datasets based on Google BigQuery platform. We should extract all patents from 2016 from that have a patent classiﬁcation (cpc) starting with G06Q30/06. Then, extract all the patents from 2016-2020 that cite these patents, and build a directed network of patent citations.

In the patents-public-data/patents/pulications table, the following attributes are related:

  • publication_number: Patent publication number (DOCDB compatible), eg: 'US-7650331-B1'
  • publication_date: The publication date.
  • cpc.code: The Cooperative Patent Classification (CPC) codes.
  • citation.publication_number: The citations of this publication.

The pseudo-code to extract the data:

SELECT pub.publication_number, cite.publication_number cite_publication_number

FROM patents-public-data.patents.publications pub, UNNEST(cpc) classification, UNNEST(citation) cite

WHERE pub.publication_date > 20160000 and classification.code like 'G06Q30/06%' and (SELECT publication_date FROM patents-public-data.patents.publications WHERE publication_number = cite.publication_number) > 20160000

Then we can get a table for this directed graph of patent citations. Each row contains two connected nodes.

5. Detect important nodes based on Page Rank¶

Based on the connection relationship between different nodes, we want to detect the most important nodes. We implement Page Rank algorithm to give every node a Page Rank score which can be used to rank the nodes. The most important (central) nodes have higher ranks and more random walks arrive them.

6. Community detection¶

We find communities in graph using Clauset-Newman-Moore greedy modularity maximization, which begins with each node in its own community and joins the pair of communities that most increases modularity until no such pair exists.

7. Practice: find the most efficient patents¶

Since there are plenty of patents, we should find a small group of patents that are efficient and we can focus on. In this way, we can save a lot of money, time and human resource. Here we use three metrics to evaluate the efficiency.

A. Cost

which is defined by the sum of the square of the number of neighbours of each node in the node list

In [21]:
def compute_cost(node_list):
    cost_cnt = 0
    for node in node_list:
        # Sum the square of number of neighbors
        cost = (len(list(network_data.neighbors(node))))**2
        cost_cnt = cost_cnt + cost
    return cost_cnt

B. Coverage

which is defined by the ratio between the number of all neighbors of the node list and the number of all nodes in the graph.

In [22]:
def compute_coverage(node_list):
    all_neighbor_nodes = []
    # Find all the neighbors
    for node in node_list:
        all_neighbor_nodes.extend([n for n in network_data.neighbors(node)])
    # Compute the ratio
    return len(set(all_neighbor_nodes))/len(network_data.nodes())

C. Closeness centrality

Closeness centrality of a node u is the reciprocal of the sum of the shortest path distances from u to all n-1 other nodes. Since the sum of distances depends on the number of nodes in the graph, closeness is normalized by the sum of minimum possible distances n-1.

The higher value indicates higher centrality.
